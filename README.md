# Desktop Client and Payment Due Date Management System

A desktop application developed to manage clients, payment records, and due dates for health insurance plans.

## Overview

This project was developed as part of an Academic Extension Project in the Systems Analysis and Development program.

The application was designed for the company Medeiros e Moreira Benefícios, aiming to improve the management of client information, payment tracking, and due date monitoring through a simple and efficient desktop solution.

The system operates primarily offline using a local database, ensuring accessibility and reliability during daily operations.

## Features

- Client registration and management
- Payment tracking and status control
- Automatic due date monitoring
- Detection of overdue payments
- Active and inactive client management
- Search and filtering tools
- Dashboard with business indicators
- Data export functionality
  - Excel (.xlsx)
  - CSV (.csv)
  - JSON (.json)
- Desktop application support
- License validation system
- Offline operation using a local database

## Technologies Used

- HTML5
- CSS3
- JavaScript (ES6+)
- Electron
- SQLite

## System Modules

### Dashboard
Provides a general overview of client information, active contracts, payment status, and overdue records.

### Client Management
Allows users to create, edit, search, and organize client information.

### Payment Management
Handles payment records, due dates, overdue calculations, and status updates.

### Data Export
Exports information to multiple formats for reporting and backup purposes.

### Licensing System
The application includes a license validation mechanism connected to an external server. Once activated, the license information is stored locally, allowing offline operation. If the license expires, access to the system is automatically blocked.

## Project Purpose

The main objective of this project is to provide a practical solution for managing health insurance clients and payment due dates while reducing manual work and improving operational organization.
